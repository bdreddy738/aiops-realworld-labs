# Interview Questions

## What is Isolation Forest?

Isolation Forest is an unsupervised anomaly detection algorithm that identifies unusual observations by isolating them through random partitioning.

---

## Why Isolation Forest?

Because it is lightweight, fast, scalable, and does not require labeled training data.

---

## What does contamination mean?

Contamination represents the expected proportion of anomalies in the dataset.

---

## Why use unsupervised learning?

Because labeled anomaly data is typically unavailable in infrastructure monitoring.

---

## Why did the model alert at 2.9% CPU?

The model compares current values against learned historical behavior rather than fixed thresholds. If 2.9% significantly deviates from historical patterns, it may be classified as an anomaly.

---

## Difference Between Threshold Monitoring and ML Monitoring?

Threshold monitoring uses fixed limits.

ML monitoring learns normal behavior and identifies unusual deviations.

---

# Summary

This project uses an Isolation Forest model to learn historical CPU behavior from Prometheus metrics. During runtime, the model evaluates real-time CPU values and classifies them as normal or anomalous. When anomalies are detected, Slack notifications are generated automatically, demonstrating a practical AIOps implementation using machine learning.
